## 🚀 建议新增功能

### 1. 角色图加载节点 ⭐⭐⭐⭐⭐

**重要性**: 非常高  
**原因**: 角色外观一致性是漫剧质量的关键

**功能设计**:
```python
class CharacterImageLoaderNode:
    """角色图加载节点"""
    
    功能:
    1. 加载角色参考图
    2. 提取角色特征
    3. 生成角色描述
    4. 维护角色一致性
    
    输入:
    - image_url: 角色图片URL
    - character_name: 角色名称
    - character_description: 角色描述（可选）
    
    输出:
    - character_data: 角色数据（包含特征、描述、seed等）
    - image_preview: 图像预览
```

**实现方式**:
- 使用图像识别API提取角色特征
- 生成角色embedding用于一致性控制
- 提供seed管理功能

### 2. 音色定义节点 ⭐⭐⭐⭐⭐

**重要性**: 非常高  
**原因**: 多角色配音需要不同音色

**功能设计**:
```python
class VoiceDefinitionNode:
    """音色定义节点"""
    
    功能:
    1. 定义角色音色
    2. 音色参数设置
    3. 音色预览
    4. 音色复用
    
    输入:
    - character_name: 角色名称
    - voice_type: 音色类型（男/女）
    - voice_style: 音色风格（温柔/激昂/低沉等）
    - voice_params: 音色参数（音调、语速等）
    
    输出:
    - voice_data: 音色数据
    - voice_preview: 音色预览音频
```

**支持的TTS引擎**:
- Minimax TTS - 中文最佳
- OpenAI TTS - 多语言
- ElevenLabs - 高质量英文

### 3. 角色管理节点 ⭐⭐⭐⭐

**重要性**: 高  
**原因**: 多角色场景需要角色管理

**功能设计**:
```python
class CharacterManagerNode:
    """角色管理节点"""
    
    功能:
    1. 注册角色（外观+音色）
    2. 角色一致性维护
    3. 角色数据库
    4. 角色复用
    
    输入:
    - character_image_data: 角色图像数据
    - character_voice_data: 角色音色数据
    - character_name: 角色名称
    
    输出:
    - character_profile: 完整角色档案
    - character_id: 角色ID（用于引用）
```

### 4. 分镜图尺寸独立设置节点 ⭐⭐⭐

**重要性**: 中  
**原因**: 分镜图可以更小以节省成本

**功能设计**:
```python
class SceneImageSizeNode:
    """分镜图尺寸设置节点"""
    
    功能:
    1. 独立设置分镜图尺寸
    2. 自动计算视频尺寸
    3. 预设尺寸模板
    
    输入:
    - scene_image_size: 分镜图尺寸
    - video_size: 视频尺寸（可选，默认自动）
    - platform: 平台类型（mobile/web/cinema）
    
    输出:
    - scene_image_size_data: 分镜图尺寸数据
    - video_size_data: 视频尺寸数据
```

**预设尺寸**:
```
分镜图（经济）: 1024x576 (节省50%成本)
视频: 1920x1080 (1080p)

分镜图（标准）: 1280x720 (720p)
视频: 1920x1080 (1080p)
```

## 💡 可借鉴的设计思路

### 1. 模块化分组设计

参考工作流使用了清晰的功能分组：
```
音频处理区
├── 台词处理
├── 加载模型-音色生成
├── 加载模型台词生成
└── 音色生成

图像处理区
├── 加载角色图
├── 提示词
│   ├── 音色/台词
│   ├── 分镜提示词
│   └── 视频提示词
├── 模型加载
│   ├── 分镜图模型
│   └── 视频模型
└── 尺寸设置
```

**我们可以借鉴**:
- 创建更清晰的节点分组
- 提供模块化的工作流模板

### 2. 变量传递系统

参考工作流使用SetNode/GetNode实现参数复用：
```
SetNode("width", 1920)
  ↓
GetNode("width") → 图像生成节点
GetNode("width") → 视频生成节点
```

**我们可以借鉴**:
- 在配置节点中提供全局参数
- 使用ComfyUI的Reroute节点

### 3. 音色数据类型

参考工作流定义了SPEAKER类型：
```
TDQwen3TtsDefineSpeaker
  → SPEAKER类型数据
  → 音色预览
  → TTS生成节点
```

**我们可以借鉴**:
- 定义CHARACTER类型（包含外观+音色）
- 定义VOICE类型（音色数据）
- 提供类型检查和预览
