## 1. 预览模式 (PreviewModeNode) 🔍

### 功能说明

生成1-3个场景的快速预览版本，用户审核满意后再生成最终版本。

### 使用场景

- ✅ 首次使用，不确定效果
- ✅ 测试新的提示词风格
- ✅ 成本敏感，避免浪费
- ✅ 快速验证场景设计

### 参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| scenes_json | STRING | 场景JSON数据 | 必填 |
| preview_count | INT | 预览场景数量 | 3 |
| preview_mode | ENUM | 预览模式 | fast |
| approved_scenes | STRING | 已批准场景ID | 可选 |

**preview_mode 选项**:
- `fast` - 快速预览（经济模型）
- `balanced` - 平衡预览（中端模型）
- `quality` - 高质量预览（高端模型）

### 使用示例

#### 步骤1: 生成预览

```
输入:
  - 场景数量: 3
  - 预览模式: fast

输出:
  - 3个场景的预览版本
  - 预览成本: $0.60 (3 × $0.20)
  - 预览摘要
```

#### 步骤2: 审核预览

查看预览结果：
- 场景1: 满意 ✅
- 场景2: 需要调整 ⏳
- 场景3: 满意 ✅

#### 步骤3: 生成最终版本

```
输入:
  - 已批准场景: 1,3

输出:
  - 场景1和3的高质量版本
  - 最终成本: $0.94 (2 × $0.47)
  - 总成本: $1.54 (预览$0.60 + 最终$0.94)
```

### 成本对比

| 方案 | 10场景成本 | 返工成本 | 总成本 |
|------|-----------|---------|--------|
| 直接生成 | $5.30 | $10.60 (2次返工) | $15.90 |
| 预览模式 | $2.00 | $3.76 (仅生成满意的) | $5.76 |
| **节省** | - | - | **64%** |

## 2. 场景编辑器 (SceneEditorNode) ✏️

### 功能说明

实时编辑场景描述、对话、情绪和视觉提示词。

### 使用场景

- ✅ AI生成的场景需要调整
- ✅ 修改对话内容
- ✅ 调整情绪氛围
- ✅ 优化视觉提示词

### 参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| scenes_json | STRING | 场景JSON数据 | 必填 |
| scene_number | INT | 要编辑的场景编号 | 1 |
| new_description | STRING | 新的场景描述 | 可选 |
| new_dialogue | STRING | 新的对话内容 | 可选 |
| new_mood | ENUM | 新的情绪标签 | 可选 |
| new_visual_prompt | STRING | 新的视觉提示词 | 可选 |

**mood 选项**:
- `romantic` - 浪漫
- `tension` - 紧张
- `happy` - 快乐
- `sad` - 悲伤
- `action` - 动作
- `neutral` - 中性

### 使用示例

#### 示例1: 修改场景描述

```
原场景:
  场景1: "林晓薇在街上行走"

修改:
  new_description: "林晓薇在春日午后的梧桐树下行走，阳光斑驳"

结果:
  ✅ 场景描述已更新
  ✅ 视觉提示词自动优化
```

#### 示例2: 调整对话

```
原对话:
  "(breath) 小心！你没事吧？"

修改:
  new_dialogue: "(担忧地) 你还好吗？有没有受伤？"

结果:
  ✅ 对话内容已更新
  ✅ 音频将重新生成
```

### 最佳实践

1. **预览后编辑** - 先生成预览，再针对性调整
2. **保持一致性** - 修改时注意角色和场景的一致性
3. **逐步优化** - 每次修改一个元素，观察效果

## 3. 失败重试机制 (RetryMechanismNode) 🔄

### 功能说明

自动检测失败任务，智能重试，提高成功率。

### 使用场景

- ✅ 网络不稳定
- ✅ API临时故障
- ✅ 资源暂时不可用
- ✅ 提高系统稳定性

### 参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| task_json | STRING | 任务JSON数据 | 必填 |
| max_retries | INT | 最大重试次数 | 3 |
| retry_delay | INT | 重试延迟(秒) | 2 |
| exponential_backoff | BOOLEAN | 指数退避 | True |

### 重试策略

#### 指数退避算法

```
第1次失败 → 等待 2秒 → 重试
第2次失败 → 等待 4秒 → 重试
第3次失败 → 等待 8秒 → 重试
```

**优势**:
- 避免过度重试
- 给服务恢复时间
- 提高成功率

### 使用示例

```
任务: 生成图像
第1次尝试: ❌ 失败 - 网络超时
等待 2秒...
第2次尝试: ❌ 失败 - 服务繁忙
等待 4秒...
第3次尝试: ✅ 成功

总耗时: 6秒 + 生成时间
成功率: 95%+
```

### 成功率提升

| 场景 | 无重试 | 有重试 | 提升 |
|------|--------|--------|------|
| 网络不稳定 | 70% | 95% | +25% |
| API故障 | 50% | 85% | +35% |
| 高峰期 | 60% | 90% | +30% |

## 4. 并行执行 (ParallelExecutionNode) ⚡

### 功能说明

并行执行多个任务，大幅缩短生成时间。

### 使用场景

- ✅ 多场景同时生成
- ✅ 视频+音频+音乐并行
- ✅ 批量处理
- ✅ 时间紧迫

### 参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| tasks_json | STRING | 任务列表JSON | 必填 |
| max_workers | INT | 最大并发数 | 3 |

### 并行策略

#### 推荐: 视频+音频+音乐并行

```
传统串行执行:
  图像 → 视频 → 音频 → 音乐
  总时间: 20分钟

并行执行:
  图像 → [视频, 音频, 音乐] (同时进行)
  总时间: 8分钟

节省时间: 60%
```

#### 并发数选择

| 并发数 | 适用场景 | 优势 | 劣势 |
|--------|---------|------|------|
| 1-2 | API限制严格 | 稳定 | 慢 |
| 3-5 | 日常使用 | 平衡 | 中等 |
| 6-10 | 高速网络 | 快速 | 可能触发限制 |

### 使用示例

```
输入:
  - 任务数: 10个场景
  - 并发数: 3

执行:
  批次1: 场景1-3 并行
  批次2: 场景4-6 并行
  批次3: 场景7-9 并行
  批次4: 场景10

总时间: ~15分钟 (vs 串行50分钟)
节省: 70%
```

## ❓ 常见问题

### Q: 预览模式和直接生成有什么区别？

A: 预览模式使用经济模型快速生成，成本低但速度快，用于验证效果。直接生成使用高质量模型，成本高但质量好。

### Q: 如何选择并发数？

A: 建议从3开始，根据API限制和网络状况调整。如果遇到频率限制错误，降低并发数。

### Q: 重试机制会影响成本吗？

A: 不会。重试失败的任务不会产生额外费用，只有成功的生成才会计费。

### Q: 质量评分准确吗？

A: 质量评分基于多个维度自动评估，准确率约85-90%，建议结合人工审核。

### Q: 如何设置预算限制？

A: 根据项目预算和场景数量计算，建议留出10-20%的缓冲空间。
